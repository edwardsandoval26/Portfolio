## Problem Statement

Radiologists spend an average of 3-4 seconds per image during high-volume screening. An AI-assisted triage system can prioritize abnormal cases, reducing missed diagnoses and improving throughput.

## Dataset

- **Source**: NIH Chest X-ray dataset (publicly available)
- **Size**: 112,120 frontal X-ray images from 30,805 patients
- **Labels**: 14 disease categories + "No Finding"
- **Split**: 70% train, 10% validation, 20% test (patient-level split to prevent data leakage)

## Model Architecture

Fine-tuned **ViT-B/16** (Vision Transformer) pretrained on ImageNet-21k:

```python
import timm

model = timm.create_model(
    'vit_base_patch16_224',
    pretrained=True,
    num_classes=15,
    drop_rate=0.1,
)

# Freeze early layers, fine-tune last 4 transformer blocks
for param in model.parameters():
    param.requires_grad = False
for block in model.blocks[-4:]:
    for param in block.parameters():
        param.requires_grad = True
for param in model.head.parameters():
    param.requires_grad = True
```

### Training Configuration

- **Optimizer**: AdamW (lr=1e-4, weight_decay=0.05)
- **Scheduler**: Cosine annealing with warmup (5 epochs)
- **Augmentation**: RandAugment, CutMix, MixUp
- **Loss**: Binary Cross-Entropy with label smoothing (0.1)
- **Training**: Mixed precision (FP16) on 2× A100 GPUs

## Engineering Details

- **Serving**: Gradio web interface for interactive demo
- **Inference**: ONNX export with TensorRT optimization
- **Infrastructure**: Docker container, deployable to AWS EC2 GPU instances
- **Reproducibility**: DVC for data versioning, Hydra for config management
- **Testing**: pytest with fixture-based testing for data pipeline
